总的来说是这样子的。
MotorCell通过内部持有&'static RefCell<T>,而RefCell::borrow()允许多个不可变借用同时存在，所以只要没有人在borrow_mut，不同地方可以同时通过borrow()读取同一个电机实例的数据。
而MotorHandle本身就是为了能被多个地方持有而设计的。多个持有者可以通过一个Channel向bus task发送命令，而channel内的fifo会自己保证执行顺序。

### 2026-5-08
虽然还没实践出昨日的构思，但是考虑到后续我调试很多地方用到的是结构体，那么简单的枚举出rust所有变量类型是不够的。
现在的想法是结构体和独立变量采用两套不同的注册方案。

结构体情况的话，类似如下
```rust
    static ARM_STATE： RefCell<ArmState> = RefCell::new(ArmState{
        target_heigh: 0.20,
        current_high: 0.0,
        status: ArmStatus::Idle,
    });
```
1. 注册表条目改造
若是想做成像keil那般展开结构体，逐个乘员读写，关键是：**注册表里存储的是成员路径，且指向的是同一个RefCell，通过函数指针实现不同字段的读写**

将每个条目从原先[指向一个独立RefCell]变成[指向一个RefCell + 字段读写函数]

同一个`RefCell<ArmState>`可以被多个条目引用，但每个条目的read/write函数读写的字段不用。借用发生在各自函数内部，不会相互干扰。

2. 宏驱动注册
手动位每个结构体写一段读写函数是很麻烦的。用`maoro_rules!`展开
```rust
watch_struct!(ArmState, &ARM_STATE, ReadWrite{
    target_heigh: f32 => ReadWrite
    current_high: f32 => ReadOnly
    status: u8 => ReadOnly,
});
```

宏展开后可以自动生成:
```rust
// 每个字段一对专用函数
fn _r_arm_target_height(ptr: *const ()) -> heapless::Vec<u8, 32> {
    let cell = unsafe { &*(ptr as *const RefCell<ArmState>) };
    let val = cell.borrow().target_height;
    f32_to_bytes(val)
}
fn _w_arm_target_height(ptr: *const (), raw: &[u8]) -> bool {
    let cell = unsafe { &*(ptr as *const RefCell<ArmState>) };
    if let Some(v) = bytes_to_f32(raw) {
        cell.borrow_mut().target_height = v;
        return true;
    }
    false
}
// ... current_height, status 同理 ...

// 注册条目
WATCH_TABLE.register(WatchEntry {
    path:   "arm.target_height",
    parent: "arm",
    ptr:    &ARM_STATE as *const _ as *const (),
    read:   _r_arm_target_height,
    write:  _w_arm_target_height,
    type_name: "f32",
    access: ReadWrite,
});
```

3.上行遥测
`debug_watch_task` 进行注册表的遍历，每个条目调用(entry.read) (entry.ptr)，拿到字节后拼上行帧：
```rust
arm.target_height = 500.0
arm.current_height=500.0
arm.status=0
chassis.left_rpm=1000.0
chassis.right_rpm=1000.0
```

然后在pc端可以按parent分组重建树：
```rust
arm ── target_height  500.0  RW 
    ├─ current_height 500.0  RO
    └─ status          0      RO
chassis ── left_rpm   1000.0  RW
        └─ right_rpm  1000.0  RW
```

4. 下行修改
PC发送对应的下行修改命令 `set xxxx value` -> 下行解析 -> 查注册表找到对应目标 -> 调用 `entry.write  entry.ptr` -> `borrow_mut()` -> 修改完成并返回`Ok`
